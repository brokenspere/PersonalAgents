Internal Logic & Data Flow

Event Payloads

Every trigger in the system is defined as an Event. A standard event payload looks like this:

{
  "market": "NYSE",
  "timestamp": "2026-06-11T13:30:00Z",
  "event_type": "market.open"
}

Enriched Payload: Output from the EnrichmentWorker, providing context-rich data ready for filtering.

{
  "source": "yahoo_finance_news",
  "timestamp": "2026-06-11T13:30:10Z",
  "items": [
    {
      "title": "Market rallies on open",
      "url": "https://finance.yahoo.com/news/...",
      "tickers": ["AAPL", "TSLA"],
      "sentiment_score": 0.85
    }
  ]
}

ScraperWorker Logic (Yahoo Finance)

Triggered by market.open and market.close.

  • Tooling: Uses BeautifulSoup for static HTML parsing. Includes a shared circuit breaker and header rotation to improve resilience against blocking.
  • Extraction: Scrapes the top headlines and the "Trending Tickers" sections independently.
  • Deduplication: Integrates a caching mechanism (e.g. Redis) to store parsed headlines and prevent redundant data from being emitted within a 24-hour window, saving downstream compute.
  • Output: Publishes raw Extraction Payloads using distinct sources (e.g., yahoo_finance_news vs yahoo_finance_trending) to the broker.

EnrichmentWorker Logic

Triggered by extraction payloads from the ScraperWorker.

  • Ticker Extraction: Applies NLP and regex-based logic to extract mentioned stock tickers from headline text.
  • Sentiment Analysis: Utilizes libraries such as VADER to score the sentiment of extracted headlines (-1.0 to 1.0).
  • Output: Transforms the raw data into an Enriched Payload and publishes it to the broker.

NotificationWorker Logic & Fault Tolerance

Triggered by enriched payloads.

  • Contextual Filtering: Evaluates incoming payloads against channel-specific rules (e.g., specific tickers or sentiment thresholds) before proceeding.
  • Formatting: Applies platform-specific formatting (e.g., Markdown for Discord, HTML for Telegram) based on the channel context.
  • Fault Tolerance (Delayed Retry): If an external API fails (e.g., rate limits, Discord downtime), the worker does NOT drop the message. It NACKs the event and re-queues it with exponential backoff. Also utilizes the shared circuit breaker pattern.
