High-Level Architecture: Scalable Agent Platform

The platform is designed around a decoupled, Event-Driven Microservices Architecture. This ensures that scaling or adding new workers requires zero modifications to existing components.

System Flow

  1. Scheduler fires a market.open or market.close event based on NYSE market hours.
  2. Message Broker receives the event and routes it to the subscribed queues.
  3. ScraperWorker receives the trigger, fetches data (News and Trending Tickers), deduplicates using a cache, and publishes raw Extraction Payload events back to the broker.
  4. EnrichmentWorker listens for extraction events, performs NLP ticker extraction and sentiment analysis, and publishes an Enriched Payload.
  5. AnalystAgent listens for enriched events, fetches real market data via yfinance, queries the Gemini LLM for insights, and publishes an Analyzed Payload.
  6. ScreenerAgent listens for market close events, fetches market data for a watchlist via yfinance, applies strict technical filtering in Python, queries the LLM for a trading plan, and publishes a Screened Payload.
  7. NotificationWorker listens for analyzed or screened events, applies channel-specific filtering rules, formats the data, and pushes the payload to external APIs.

Core Components

1. Scheduler (Celery Beat / EventBridge)

A standalone component responsible purely for time-based triggers. It emits Events (JSON payloads containing at minimum an ISO-8601 timestamp and context identifiers like {"market": "NYSE"}).

2. Message Broker (Redis / SQS)

The central nervous system of the platform. All communication between components goes through the broker, ensuring workers are completely isolated from one another.

3. Workers

Independent components that consume from or publish to the message broker. Each worker is containerized/deployed independently.

  • ScraperWorker: Uses BeautifulSoup to extract headlines and applies deduplication logic to prevent redundant processing.
  • EnrichmentWorker: A new processing step that extracts ticker entities and scores headline sentiment, outputting an Enriched Payload.
  • AnalystAgent: A specialized agent that fetches market data via yfinance and queries the Gemini LLM to derive financial insights and impact scores.
  • NotificationWorker: Handles channel-specific filtering rules and API communication with Telegram and Discord.
4. Shared Codebase

Common utilities like circuit breakers, configurations, and data models are shared across independent Worker containers by utilizing Docker build contexts (copying a shared/ directory from the monorepo root), rather than relying on private PyPI packages.
