AWS Deployment Plan (Terraform)

To maximize cost optimization for this personal project, the platform will utilize an AWS Native Serverless architecture. Infrastructure as Code (IaC) will be managed via Terraform.

Target Architecture

Compute & Scheduling

AWS Lambda. Serverless compute functions for the Scraper and Notification logic. We only pay for the exact milliseconds of execution time.

Amazon EventBridge. Replaces the Celery Beat scheduler. Triggering the Scraper Lambda based on cron schedules (NYSE market hours).

Message Broker

Amazon SQS. Replaces Redis/Celery queues. The Scraper Lambda pushes extracted news payloads to an SQS queue, which triggers the Notification Lambda.

Networking

Default AWS Networking. Lambdas will run without a custom VPC, allowing them direct outbound internet access to scrape Yahoo Finance and hit the Telegram API without the cost of NAT Gateways.

Secrets Management

AWS Systems Manager (SSM) Parameter Store. Securely stores API keys and Telegram webhooks using SecureString (KMS encryption) for zero cost.

Logging & Monitoring

Amazon CloudWatch Logs. Lambda execution logs are sent here. To strictly optimize costs and avoid long-term storage fees, a strict 1-day retention policy will be applied to all log groups.

Terraform Plan Structure

terraform/
├── eventbridge.tf  # Cron schedule to trigger scraper
├── sqs.tf          # Queue connecting scraper and notification
├── lambda.tf       # Scraper and Notification functions
├── iam.tf          # Execution roles and permissions
├── ssm.tf          # SSM Parameter Store configuration
└── cloudwatch.tf   # Log groups with 1-day retention

Deployment Strategy

  1. Package & Deploy: CI/CD pipeline packages the Python code into deployment zips (or container images if dependencies require it) and updates the Lambda functions.
  2. Terraform Apply: Terraform provisions the EventBridge rules, SQS queues, Lambda functions, and necessary IAM bindings.
